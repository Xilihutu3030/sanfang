## 幻灯片2：建设背景与痛点

### 当前三防工作面临的挑战

1. **数据滞后、精度不足**
   - 现有系统数据更新慢，街镇一级难获取精准预报
   - 多源数据分散，缺乏整合

2. **缺乏地形判断能力**
   - **没有地形高程数据，无法判断哪里会淹、淹多深**
   - 研判主要靠经验，科学性不足

3. **基层工具缺失**
   - 街镇一级缺少简单好用的辅助工具
   - 圈选任意区域分析的能力缺失

4. **人工研判效率低**
   - 研判依赖人工，时间长、主观性强
   - 指挥建议不直观、不可量化

## 幻灯片3：系统定位

### 一套区/街镇/重点区域通用的三防辅助决策系统

**核心特点：**

✅ **自由圈选任意区域**（不限于行政区划）  
✅ **接入真实地形图 + 高程数据**（判断淹水核心能力）  
✅ **五级气象自动抓取**（中央+省+市+区+民间）  
✅ **AI 智能研判**（输出风险等级、危险点位、指挥建议）  
✅ **纯公开数据**（个人/单位均可落地，无涉密风险）

**适用场景：**
- 区级三防指挥中心
- 街道/镇应急办
- 重点片区/社区防灾

## 幻灯片4：核心功能（6大模块）

### 1. GIS 地图 + 地形图
- 基础地图（道路、地名、区划）
- **地形高程图**（颜色着色、等高线）
- **低洼易涝区高亮**
- 支持：框选、手绘、点选

### 2. 五级气象数据自动抓取
- 中央气象台 + 省市区气象局 + 民间气象（和风/彩云）
- 每 5 分钟自动更新
- 包含：实时雨量、预警信号、未来预报、雷达回波

### 3. 地形高程分析（本系统独有）
- 加载公开 DEM 高程数据
- 自动识别：低洼区、河道、隧道、桥下
- 输出：区域最低点高程、易涝程度评分

### 4. 本地隐患点管理
- 易涝点、危房、地下车库、河道、堤坝
- 手动录入 + 地图标注
- 历史灾情关联

### 5. AI 智能研判（核心算法）
**输入**：降雨预报 + 地形高程 + 隐患点 + 历史灾情  
**输出**：
- ① 综合风险等级（低/中/高/极高）
- ② 主要风险类型（内涝/洪水/地质）
- ③ Top5 最危险点位（含隧道、桥洞、低洼区）
- ④ 可能淹没范围描述
- ⑤ 灾害持续时间预估

### 6. 自动指挥建议 + 简报生成
- 重点巡查点位
- 隧道/桥洞封闭建议
- 低洼小区转移提示
- 抢险力量布防方案
- 一键复制简报（可直接发工作群）

## 幻灯片10：技术架构与部署

### 技术栈
- **前端**：HTML5 + JavaScript + 高德地图 API
- **后端**：Python 3.8+（爬虫 + 分析 + AI）
- **数据库**：JSON / SQLite（可扩展为 MySQL）
- **部署**：本地/云服务器均可

### 部署方式

**方案1：本地部署（适合个人/小范围试点）**
- 一台电脑即可运行
- 双击启动，浏览器访问
- 成本：**零**

**方案2：云服务器部署（适合区级/街镇级应用）**
- 部署到轻量云服务器
- 支持多终端访问（电脑/手机/平板）
- 成本：300-500元/年

**方案3：对接现有平台（适合已有应急平台的单位）**
- 作为功能模块集成到现有系统
- API 接口对接
