## 系统概述

### 什么是三防智能研判系统？

这是一套**基于 GIS 地图 + 地形高程 + 多级气象数据 + AI 智能分析**的三防辅助决策工具。

**核心能力：**
- 📍 在地图上圈选任意区域（区/街镇/自定义片区）
- 🗺️ 叠加显示地形高程图（判断哪里会淹的关键）
- 🌤️ 自动抓取五级气象数据（每5分钟更新）
- 🤖 AI 智能研判，输出风险等级和指挥建议
- 📋 一键生成简报，可直接使用

### 适用人群

- ✅ 区/县应急管理人员
- ✅ 街道/镇三防办工作人员
- ✅ 社区/村防灾减灾负责人
- ✅ 应急指挥中心值班员
- ✅ 相关技术研究人员

## 快速开始

### 方法一：一键启动（推荐新手）

1. **确保已安装 Python 3.8+**
   - 检查方法：打开命令行，输入 `python --version`
   - 如未安装，访问：https://www.python.org/downloads/

2. **双击运行**
   ```

   ```

3. **等待自动完成**
   - 自动安装依赖包
   - 自动启动爬虫
   - 自动打开网页

4. **开始使用**
   - 浏览器会自动打开系统页面
   - 等待地图加载完成

#### 第1步：安装依赖
```bash
pip install -r requirements.txt
```

```bash
python terrain.py
```

```bash
python ai_judge.py
```

## 详细功能说明

### 一、地图操作

#### 1.1 基础操作

**地图缩放：**
- 鼠标滚轮：放大/缩小
- 点击右上角按钮：🔍 放大 / 🔍 缩小

**地图移动：**
- 鼠标左键拖动

**地图重置：**
- 点击 📍 重置 按钮，回到默认视角（广州市）

#### 1.2 区域圈选（核心功能）

系统提供 3 种圈选方式：

**① 矩形框选**
```
1. 点击"📐 矩形框选"按钮
2. 在地图上点击起点
3. 拖动鼠标到终点
4. 松开鼠标，完成圈选
5. 左侧显示：✅ 已选区域约 X.XX km²
```

**② 手绘区域**
```
1. 点击"✏️ 手绘区域"按钮
2. 在地图上点击多个点，形成多边形
3. 双击完成绘制
4. 自动计算区域面积
```

**③ 圆形范围**
```
1. 点击"⭕ 圆形范围"按钮
2. 在地图上点击圆心
3. 拖动设置半径
4. 松开完成
```

**注意事项：**
- 每次只能选择一个区域
- 新圈选会覆盖旧的区域
- 圈选后才能进行智能研判

#### 1.3 地形图层控制（独家功能）

**开关说明：**

| 图层 | 说明 | 建议 |
|------|------|------|
| 高程地形图 | 颜色着色，深色=低洼 | ✅ 必开 |
| 等高线 | 显示等高线 | ✅ 建议开 |
| 低洼易涝高亮 | 红色高亮低洼区 | ✅ 必开 |
| 河道水系 | 显示河流 | ⚠️ 按需开 |

**操作方法：**
- 点击左侧开关即可开启/关闭
- 蓝色=开启，灰色=关闭

**地形判读技巧：**
- **深色区域** = 地势低洼，容易积水
- **浅色区域** = 地势较高，相对安全
- **沿河区域** + **低洼** = 最危险

### 二、气象数据

#### 2.1 数据来源（五级）

系统自动抓取以下数据源：

1. **中央气象台**（国家级）
   - 预警信号、雷达回波
   
2. **省气象局**（省级）
   - 区域预警、降雨预报

3. **市气象局**（市级）
   - 本地预报

4. **区县气象局**（区级）
   - 区域精细预报

5. **民间气象**（街镇级）
   - 和风天气、彩云天气
   - 最精细，可到街镇

#### 2.2 数据内容

左侧"🌤️ 五级气象数据"面板显示：

```
🔴 预警：暴雨蓝色预警
💧 1小时雨量：2.5 mm
☔ 24小时雨量：35.0 mm
📊 6小时预报：大雨
🕐 更新：2026-03-02 10:30:15
```

**数据说明：**
- **预警信号**：蓝色 < 黄色 < 橙色 < 红色
- **1小时雨量**：实时降雨强度
- **24小时雨量**：累计雨量（判断淹水的关键）
- **6小时预报**：未来短期趋势

#### 2.3 手动刷新

- 点击"🔄 刷新气象"按钮
- 立即抓取最新数据
- 正常情况下，系统每 5 分钟自动刷新

### 三、AI 智能研判（核心功能）

#### 3.1 触发研判

**前提条件：**
1. ✅ 已圈选区域
2. ✅ 气象数据已加载

**操作步骤：**
```
1. 在地图上圈选目标区域
2. 点击"🤖 开始智能研判"按钮
3. 等待2-3秒（显示"智能研判中..."）
4. 右侧面板显示研判结果
```

#### 3.2 研判结果解读

**① 综合风险等级**

显示为彩色卡片：

| 等级 | 颜色 | 分数 | 含义 | 响应建议 |
|------|------|------|------|----------|
| 极高风险 | 🔴 红色 | 70-100 | 必定发生严重灾害 | I级响应，立即行动 |
| 高风险 | 🟠 橙色 | 50-69 | 很可能发生灾害 | II级响应，重点防范 |
| 中风险 | 🟡 黄色 | 30-49 | 可能发生灾害 | III级响应，加强监测 |
| 低风险 | 🔵 蓝色 | 0-29 | 一般性防范 | IV级响应，常规巡查 |

**计算逻辑：**
```
风险得分 = 气象风险（40分）+ 地形风险（30分）
          + 隐患点风险（30分）
```

**② 主要风险类型**

系统会列出 1-3 种主要风险：

- **城市内涝**：低洼区 + 大雨
- **河道洪水**：沿河 + 暴雨
- **隧道/地下空间淹水**：地下 + 积水
- **地质灾害**：山区 + 暴雨

**③ Top5 最危险点位**

按风险从高到低排序：

```
1. 🚇 XX隧道入口 - 低洼+地下，极易积水
2. 🌉 XX桥洞下方 - 历史易涝点
3. 🏘️ XX老旧小区 - 排水系统老化
4. 🅿️ XX地下车库 - 地势低+排水差
5. 🛣️ XX沿河路段 - 河道水位上涨风险
```

**点位评分依据：**
- 隧道/地下：+30分
- 历史淹水：+20分
- 极低洼（<30m）：+25分
- 沿河：+20分

**④ 淹没预判（基于地形）**

```
• 区域最低点高程：28米
• 平均高程：45米
• 低洼易涝区：3处
• 可能淹没范围：约0.8平方公里
• 积水最深处预估：0.5-1.2米
```

**判断逻辑：**
- 最低点 < 30m + 雨量 > 50mm → 极易淹
- 预估积水深度：
  ```
  深度 = f(最低点高程, 降雨量, 排水能力)
  ```

**⑤ 指挥建议（可直接下发）**

系统自动生成 5-7 条建议：

```
立即行动：
1️⃣ 加强XX隧道、XX桥洞巡查，必要时封闭
2️⃣ 通知XX小区做好防范，地下车库转移车辆
3️⃣ 沿河路段设警示标志，禁止通行
4️⃣ 抢险队伍前置至XX街道待命
5️⃣ 密切监控气象预警变化，随时调整响应等级
```

**⑥ 领导汇报话术**

系统生成标准汇报语言（黄色高亮）：

```
根据智能研判，当前高风险，主要风险为城市内涝。
重点关注XX隧道、XX桥洞等5处危险点位，
已部署巡查力量，建议提升响应等级。
```

## 常见问题

### Q1：地图显示"地图加载中"，一直不动？

**原因：** 高德地图 KEY 未配置或无效

**解决方法：**
1. 打开 `index.html`
2. 找到第 6 行：
   ```html
   <script src="https://webapi.amap.com/maps?v=2.0&key=YOUR_AMAP_KEY"></script>
   ```
3. 替换 `YOUR_AMAP_KEY` 为你的真实 KEY
4. 获取 KEY：https://lbs.amap.com/ （免费注册）

---

### Q2：气象数据显示"加载中..."？

**原因：** 爬虫未启动或网络问题

**解决方法：**
1. 检查爬虫是否运行：
   ```bash
   python crawler.py
   ```
2. 检查网络连接
3. 查看 `weather_data.json` 是否存在

---

### Q3：地形高程数据获取失败？

**原因：** 高程 API 访问超时（国外服务器）

**解决方法：**
1. 多试几次（有时网络波动）
2. 或使用备用 API（需配置密钥）
3. 或使用本地 DEM 数据文件

---

### Q4：智能研判结果不准确？

**原因：** 本地隐患点数据不完整

**解决方法：**
1. 补充录入本地隐患点
2. 添加历史灾情数据
3. 调整风险阈值参数

---

### Q5：能在手机上用吗？

**答：** 可以，但推荐电脑端

- ✅ 网页支持响应式布局
- ✅ 手机可查看研判结果
- ⚠️ 圈选操作不太方便

## 高级配置

### 配置1：修改目标区域

编辑 `crawler.py`：
```python
TARGET_AREA = "深圳市"  # 改成你的区域
```

### 配置2：修改更新频率

编辑 `crawler.py`：
```python
UPDATE_INTERVAL = 3  # 改成3分钟更新
```

### 配置3：修改风险阈值

编辑 `terrain.py`：
```python
RISK_THRESHOLDS = {
    "低洼线": 45,      # 原50米，改成45米
    "极低洼": 25,      # 原30米，改成25米
}
```

### 配置4：对接本地数据库

编辑 `ai_judge.py`，添加数据库连接：

```python
import pymysql

def load_hazards_from_db():
    conn = pymysql.connect(
        host='localhost',
        user='root',
        password='password',
        database='sanfang'
    )
    cursor = conn.cursor()
    cursor.execute("SELECT * FROM hazard_points")
    return cursor.fetchall()

# 使用数据库数据
hazards = load_hazards_from_db()
```

## 最佳实践

### 场景1：日常值班使用

**操作流程：**
```
1. 上班时打开系统，检查气象数据
2. 如有预警，立即圈选重点区域
3. 执行智能研判
4. 将结果报给领导
5. 按建议部署巡查力量
```

**时间：** 5-10分钟

---

### 场景2：暴雨预警响应

**操作流程：**
```
1. 接到预警 → 立即打开系统
2. 圈选全区/全街道
3. 查看地形图，识别低洼区
4. 智能研判 → Top5 危险点位
5. 生成简报，上报 + 下发
6. 按建议封闭隧道/转移车辆
7. 每小时刷新一次，动态调整
```

---

### 场景3：台风前研判

**操作流程：**
```
1. 台风前 3 天启动系统
2. 圈选沿海/沿河区域
3. 模拟不同降雨量场景：
   - 50mm → 中风险
   - 100mm → 高风险
   - 150mm → 极高风险
4. 制定分级响应预案
5. 前置抢险力量到 Top5 点位
6. 台风登陆前 6 小时再次研判
```

---

### 场景4：灾后复盘

**操作流程：**
```
1. 查看历史研判记录
2. 对比实际灾情
3. 分析：
   - 哪些点位预判准确？
   - 哪些漏判/误判？
4. 优化：
   - 补充隐患点
   - 调整风险阈值
   - 更新地形数据
5. 总结经验，形成文档
```
